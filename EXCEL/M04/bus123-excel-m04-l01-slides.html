Header Row

Descriptive labels in Row 1. Always start with a letter — never a number.

No Blank Rows

A blank row tells Excel the table is over. Keep your data contiguous.

Consistent Types

Whole column = one data type. Don't mix currency and text in column C.

Format as Table

Getting Data In · Step 2

Flash Fill — Excel Reads Your Mind

Harborside's staff roster came in as a single "First Last" column. Finance needs first names and last names separated for payroll export.

A — Full Name B — First Name C — Last Name
1Priya NairPriya← type "Priya"
2James KowalskiJames
3Celia TorresCelia
// Keyboard shortcut
Ctrl + E // OR: Home → Fill → Flash Fill
🔍
How It Works
Type one or two examples in the adjacent column. Flash Fill detects the pattern and fills the rest automatically — no formula needed.
⚠️
When To Use Functions Instead
If your data is inconsistent (some entries "Last, First" and some "First Last"), use LEFT / MID / FIND functions for reliable extraction every time.

Getting Data In · Two More Tools

Text to Columns & Data from Picture

Text to Columns
Split One Column Into Many

When a CSV or paste dumps everything into column A, use this wizard to split it apart.

Data → Data Tools → Text to Columns 1. Choose Delimited (comma, tab, pipe)
2. Choose Fixed Width (if aligned)
3. Set column data types → Finish
Example: "Nair, Priya, RN, 40hrs" → split at commas into 4 columns.
Data from Picture
OCR → Spreadsheet in Seconds

Got a printed report or a screenshot of a table? Excel reads it.

Data → Get & Transform → From Picture // From File — saved .jpg / .png
// From Clipboard — screen capture
Requires Microsoft 365 subscription
Saves the scan-type-check cycle entirely — straight to editable cells.
🧹
Clean data in = clean charts out.
Every extra minute spent structuring data here saves five minutes of debugging a broken chart later.

Chart Selection Framework

Ask One Question: What Am I Showing?

If you want to show… Use this chart Why it works Harborside example
Comparing values across categories Column or Bar Height/length = magnitude; human eye reads these instantly Revenue by payer category (Medicare vs. Medicaid)
Change over time (continuous) Line Chart Connects dots to show trends and momentum Monthly ED visits over 12 months
Parts of a whole (2–4 pieces) Pie or Donut Shows proportional contribution — always add data labels % share of revenue by payer type
Relationship between two variables XY Scatter Only chart that spaces X-axis by actual numeric value Staffing hours vs. patient satisfaction score
Multi-category hierarchy Treemap Nested rectangles — human eye compares area, not arcs Revenue by department → by billing code

Chart Type · Line Chart

Trends & Change Over Time

When to use a Line Chart
  • Showing values over time (months, quarters, years)
  • Tracking trends — is it rising, falling, or flat?
  • Comparing multiple trends on one chart
Harborside scenario: The CFO wants to see ED visit volume by month over the past year — are summer months driving a spike?
⚠ Line vs. Scatter Warning

A Line chart spaces X-axis points equally even if values aren't. If your X-axis has unequal numeric intervals (100, 200, 500, 1000), use an XY Scatter to avoid misleading the viewer.

Chart Type · Pie & Donut

Showing Parts of a Whole

Pie Chart Rules
  • Use for 2–4 categories only — more slices = unreadable arcs
  • Always add data labels — the eye can't judge arc size accurately
  • Donut charts free up the center for a KPI or annotation
Harborside scenario: What percentage of total Q3 revenue comes from each payer category? 4 categories = perfect for a pie.
Insert → Charts → Pie → 2D Pie → Add Data Labels → Show Percentage

COMMON MISTAKE · EXCEL-M04-L01
THE MISTAKE

Using a Pie Chart for Too Many Categories

A student builds a pie chart with 9 payer sub-categories. The chart becomes a ring of tiny wedges — no one can read the labels, and the 80/20 insight is completely buried.

ALSO WATCH FOR: Using a Line chart for category data (Revenue by Department) instead of a Column chart.
THE FIX

Consolidate or Switch Chart Type

  • Group smaller categories into "Other"
  • If you have 5+ categories, switch to a Clustered Column chart
  • Add data labels to every pie/donut — always
  • Ask: "Does this visual answer the question?" If not, pick a different chart

Key Takeaways

What Sticks Today

1
Clean data first, chart second. A well-structured Excel Table with no blank rows and consistent data types is the prerequisite for every chart.
Flash Fill · Text to Columns · Data from Picture are your ingestion shortcuts.
2
Chart type is a logic choice, not a style choice. Column = compare categories. Line = track trends. Pie = show proportions (2–4 slices max). Scatter = compare two numeric variables.
Wrong chart type = misleading story. Always ask "what am I showing?"
3
Conditional Formatting is a visual audit, not just decoration. Apply it to catch outliers, flag errors, and make key numbers jump out before your work goes to a stakeholder.
Home → Styles → Conditional Formatting → Color Scales / Icon Sets.
